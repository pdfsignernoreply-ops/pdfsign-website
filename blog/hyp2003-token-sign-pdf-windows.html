How to Use HyperPKI (HYP2003) Token to Sign PDFs on Windows

By PDFSign  ·  Aug 2025  ·  5 min read
Home › Blog › How to Use HyperPKI (HYP2003) Token to Sign PDFs on Windows

The HYP2003 is one of the most widely used DSC token hardware devices in India. It stores certificates issued by Certifying Authorities including eMudhra, Capricorn, NSDL, (n)Code, Pentasign, vsign, and Sify. Despite being common, users frequently report that the HYP2003 is not detected by PDF signing software.

Why HYP2003 Tokens Go Undetected

Most PDF tools look for certificates in the Windows Certificate Store. The HYP2003 stores certificates in PKCS#11 format directly on the hardware — not in the Windows store. Any tool that does not support PKCS#11 will show an empty certificate list even when your token is plugged in and working correctly.

PDFSign software reads certificates directly from PKCS#11-compatible tokens including the HYP2003, without requiring manual import into Windows.

Step-by-Step: Sign PDFs with HYP2003

1Download and Install the HyperPKI Driver

This is mandatory. Without the driver, Windows cannot communicate with your HYP2003 token and no software will detect it. Download the official HyperPKI middleware: HyperPKI-HYP2003-Middleware-Windows-v1.1.25.731.zip. Install it and restart your computer if prompted. The driver installs as HYPER_PKI HYP 2003 Token Manager in your Start menu.

2Plug In Your HYP2003 Token

Insert the token into a USB port. Wait for Windows to detect it — you may see a notification in the system tray. If the device shows an error in Device Manager, the driver installation may need to be repeated.

3Open PDFSign Software

Launch PDFSign from your desktop or Start menu.

4Select Your Certificate

PDFSign automatically scans connected PKCS#11 tokens and displays your certificate in the dropdown. If multiple certificates are present — for example if you have multiple tokens connected — click Refresh. Select your certificate and verify the name matches your expected identity.

5Select Signing Mode and Load Your Document

Choose Single PDF to sign one document and click Browse to select the file. Choose Batch mode to sign multiple PDFs at once — select all the files you want to process. PDFSign will use the same certificate and settings for each file.

6Place Your Signature

Drag your signature box directly on the PDF preview, or enter page and X/Y coordinate values in the fields for precise placement.

7Choose Signing Scope

Select Single Page or All Pages as required by the document.

8Set Password Protection (Optional)

Configure password protection or document permissions in the Document Security section if needed before signing.

9Click Sign and Enter Your PIN

Click Sign. Enter your HYP2003 token PIN when prompted. PDFSign applies your digital signature.

10Collect Your Signed File

The signed PDF is saved as filename_signed.pdf in the same folder. For batch signing, each document is saved individually with the _signed suffix. Change the output location before signing if a different folder is required.

ProxyKey note: PDFSign also handles ProxyKey (Watchdata) tokens where the Common Name sometimes shows as "localhost" — the app correctly extracts your identity from the Subject Organisation field.

Frequently Asked Questions

My HYP2003 token is plugged in but not showing in PDFSign. What do I do?

Click the Refresh button in PDFSign. If still not detected, check that the HyperPKI driver is installed correctly — open HYPER_PKI HYP 2003 Token Manager from the Start menu and verify the token appears without errors. If not, reinstall the HyperPKI driver and restart your PC.

Can I use HYP2003 with a certificate from any CA?

Yes. The HYP2003 is a hardware token that can store certificates from any licensed CA — eMudhra, Capricorn, NSDL, (n)Code, Pentasign, vsign, or Sify. PDFSign supports all of these.
