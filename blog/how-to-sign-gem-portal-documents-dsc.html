How to Sign Documents on GeM Portal Using DSC

By PDFSign  ·  Oct 2025  ·  5 min read
Home › Blog › How to Sign Documents on GeM Portal Using DSC

The Government e-Marketplace (GeM) requires all registered sellers and service providers to digitally sign documents before submission. Without a valid DSC-signed PDF, your bid or order confirmation is incomplete and may be rejected outright.

Most users try signing through browser plugins that conflict with Windows security policies. The result is a document that fails validation or carries no visible signature. The correct approach: sign on your desktop using PDFSign software, then upload the signed file to GeM.

What You Need

  • Windows PC with PDFSign software installed (30-day free trial)
  • USB DSC token — HYP2003 or ProxyKey (Watchdata)
  • Token driver installed — use the driver for your specific token brand (HYP2003, ProxyKey, Gemalto, mToken, etc.)
  • Class 3 DSC issued by a licensed CA: eMudhra, Capricorn, NSDL, (n)Code, Pentasign, vsign, or Sify

Common Certificates Used for GeM

Certificate TypeUse Case
Class 3 Individual DSCIndividual sellers, freelancers
Class 3 Organisation DSCCompany/firm registration and bids

Step-by-Step: Sign GeM Documents

1Install Your Token Driver

Mandatory before anything else. Install the driver/middleware for whichever token you have — HYP2003 (HyperPKI), ProxyKey (Watchdata), Gemalto, mToken, etc. Download it from your token vendor's website or from your CA's support page. Restart your PC if prompted.

2Plug In Your DSC Token

Insert your USB token and wait for Windows to detect it.

3Open PDFSign Software

Launch PDFSign from your desktop or Start menu.

4Select Your Certificate

PDFSign automatically detects your token and shows your certificate in the dropdown. If you have multiple certificates, click Refresh and select your Class 3 certificate. Verify the name matches your GeM-registered identity.

5Select Signing Mode and Load Your Document

For a single GeM document, choose Single PDF mode and click Browse to load the downloaded document. For multiple GeM documents to sign in one session, use Batch mode and select all files together.

6Place Your Signature

Drag your signature box on the PDF preview to the required signature location, or enter the page number and position values manually. Refer to the GeM document for where the signature should appear.

7Choose Signing Scope

Select Single Page or All Pages depending on the document requirement.

8Click Sign and Enter Your PIN

Click Sign. Enter your token PIN when prompted. PDFSign applies your digital signature.

9Save and Upload to GeM

The signed file is saved automatically as filename_signed.pdf. Set a custom output folder before signing if needed. Upload the signed PDF back to GeM to complete your submission.

Important: Always fill in all required fields in the GeM document before signing. Any modification after signing will break the cryptographic seal and invalidate the signature.

Frequently Asked Questions

Why does GeM not accept my signature?

The most common reasons are: using an expired DSC, a signature applied as an image rather than a cryptographic digital signature, or the wrong certificate selected during signing. GeM requires Class 3 DSC — PDFSign applies a cryptographically valid PAdES signature that GeM's verification system accepts. Note: Class 2 DSCs are no longer issued in India, so if your token has a valid certificate it is already Class 3.

Can I sign multiple GeM documents at once?

Yes. Use PDFSign's Batch mode to select and sign multiple documents in one session with the same certificate and settings.

Which CAs issue valid DSCs for GeM?

eMudhra, Capricorn, NSDL, (n)Code, Pentasign, vsign, and Sify are all licensed Certifying Authorities whose Class 3 DSCs are accepted on GeM portal.
