DSC Token Not Detected on Windows 11 — Complete Fix Guide

By PDFSign  ·  April 2026  ·  6 min read
Home › Blog › DSC Token Not Detected on Windows 11 — Complete Fix Guide

Your USB DSC token worked perfectly on Windows 10. You upgraded to Windows 11 — or bought a new laptop that came with Windows 11 — and now your token either isn't detected at all or appears in Device Manager but your signing software shows no certificate. This is a common problem, and it has specific, fixable causes.

This guide covers every known fix for HYP2003 (HS series), ePass 2003/3003, Proxkey (WD series), Feitian, Watchdata, and WD Pro tokens on Windows 11, including 64-bit Home and Pro editions.

Why Windows 11 Causes More Detection Issues Than Windows 10

Windows 11 introduced several changes that directly affect DSC token detection:

  • Stricter Driver Signature Enforcement (DSE): Windows 11 requires all kernel-mode drivers to be digitally signed by Microsoft. Older token drivers that were never updated for this requirement will fail to load silently.
  • Aggressive USB Power Management: Windows 11's default power plan puts USB ports into selective suspend more aggressively, causing tokens to disconnect or fail to initialise after the PC wakes from sleep or idle.
  • Secure Boot compatibility: On some systems, Secure Boot interactions with the UEFI firmware affect which drivers can load during startup, impacting middleware that uses early-load mechanisms.
  • 64-bit requirement: Windows 11 is 64-bit only. Some legacy 32-bit token middleware installers do not install correctly on Windows 11, even if they ran on 64-bit Windows 10.
Common symptoms: Token LED blinks when inserted but PC does not detect it / token manager software shows "No token found" or an empty certificate list / Device Manager shows the device with a yellow exclamation mark / works sometimes but disappears after sleep.

Fix 1: Install the Correct Driver for Your Token

1Remove the Token First

Unplug the DSC token from your PC before installing any driver. Installing with the token already connected can cause conflicts during driver setup.

2Download the Correct Middleware

For HYP2003 (HS series): download the ePass2003 Auto PKCS#11 driver. For Proxkey (WD series): visit proxkey.com and download the Proxkey middleware installer. For ePass 3003: visit etokenindia.com. For Feitian: visit ftsafe.com and download SafeSign Identity Client.

3Run the Installer as Administrator

Right-click the downloaded installer and select "Run as administrator." Do not double-click directly. Administrator rights are mandatory for driver installation on Windows 11.

4Restart Your PC

After installation completes, restart Windows. Do not plug in the token yet.

5Plug in the Token

After the PC has fully restarted, plug in your DSC token. Windows should now detect it and load the correct driver automatically.

6Open Token Management Software

Open the token management software that came with your middleware (e.g., ePass2003 Token Manager or Proxkey Manager). You should see your certificate listed. If yes, proceed to signing.

Fix 2: Disable USB Selective Suspend

This is one of the most common causes of intermittent token detection on Windows 11 — the token disappears after the PC is idle for a few minutes.

1Open Power Options

Press Win + R, type powercfg.cpl and press Enter. Or search for "Power Options" in the Start menu.

2Change Plan Settings

Click "Change plan settings" next to your active power plan, then click "Change advanced power settings."

3Find USB Settings

Expand USB settings > USB selective suspend setting. Change the setting to Disabled for both "On battery" and "Plugged in."

4Apply and Restart

Click Apply, then OK. Restart your PC. Your token should now remain detected even after idle periods.

Fix 3: Always Run Signing Software as Administrator

On Windows 11, PKCS#11 middleware often requires administrator privileges to access hardware cryptographic keys. If your token is detected by the management software but PDFSign shows no certificate, try running PDFSign as administrator.

Right-click the PDFSign shortcut on your desktop or in the Start menu and select "Run as administrator." To make this permanent, right-click the shortcut, choose Properties, click the Compatibility tab, and check "Run this program as an administrator."

Fix 4: Check Device Manager for Driver Errors

1Open Device Manager

Press Win + X and select Device Manager from the menu.

2Locate the Token

Plug in your token and look for it under "Smart card readers" or "Universal Serial Bus devices." A yellow exclamation mark indicates a driver problem.

3Uninstall the Device

Right-click the device with the exclamation mark and select "Uninstall device." Check the box to delete the driver software if prompted.

4Reinstall the Driver

Unplug the token, run the correct middleware installer as administrator again, restart, and plug the token back in.

Fix 5: Disable Driver Signature Enforcement (Last Resort)

Only use this fix if you have confirmed that your token driver is an older version that is not signed for Windows 11, and no updated version is available from the vendor.

Warning: Disabling driver signature enforcement reduces system security. Only do this temporarily during driver installation and re-enable it afterwards. Do not leave it disabled permanently on production systems.
1Open Advanced Startup

Go to Settings > System > Recovery and click "Restart now" under Advanced startup.

2Choose Troubleshoot

On the blue screen, choose Troubleshoot > Advanced options > Startup Settings > Restart.

3Select Option 7

After restart, press 7 or F7 to select "Disable driver signature enforcement." Windows will start with enforcement disabled for this session only.

4Install the Driver

Install your token middleware during this session. The unsigned driver will be allowed to install. Restart normally after installation — enforcement will re-enable automatically.

Fix 6: Use a Direct USB Port — Avoid Hubs

USB hubs — especially unpowered hubs — frequently cause intermittent detection issues with cryptographic tokens. The token requires a stable power supply and USB signal integrity that many cheap hubs cannot provide.

Always plug your DSC token directly into a USB port on the computer chassis (the ports built into the laptop body or the back panel of a desktop). If you are using a laptop docking station, try plugging directly into the laptop instead. Front panel USB ports on desktops can also be unreliable — try the rear panel ports first.

Tip: If using a laptop, avoid plugging the token into the same USB controller as other high-bandwidth devices (external hard drives, webcams). DSC tokens are low-bandwidth but sensitive to USB bus congestion.

How to Confirm Your Token Is Working Correctly

Before attempting to sign any document, verify that your token is fully operational at the middleware level:

  1. Open your token management software (ePass2003 Manager, Proxkey Manager, or equivalent)
  2. You should see your token listed with its serial number
  3. Click on the token to expand the view — you should see at least one certificate listed (Digital Signature or Encryption)
  4. Try changing the PIN through the management software — if it succeeds, the token is fully accessible

If the management software cannot see the token at this stage, the problem is at the driver or hardware level — go back through Fixes 1 through 6. If the management software sees the certificate but PDFSign does not, ensure PDFSign is running as administrator.

Frequently Asked Questions

My token worked on Windows 10 but not on Windows 11 after upgrade — why?

Windows 11 enforces stricter driver signature requirements. Older token drivers that were not updated for Windows 11 compatibility will fail to load. The fix is to install the latest driver version from your token vendor's website. In most cases, updated drivers are available — the driver you installed years ago on Windows 10 is simply not compatible with Windows 11's kernel requirements.

My token appears in Device Manager but the token management software shows nothing

Device Manager detects the USB hardware, but the PKCS#11 middleware layer — which reads the certificate stored on the token's smart card chip — may not be installed or may be outdated. Uninstall the current middleware completely (via Add/Remove Programs), restart, then reinstall the latest version from the vendor's website as administrator.

I entered the wrong PIN too many times and now the token is locked

Contact your DSC issuing Certifying Authority or the token vendor. Most tokens can be unlocked using a PUK (PIN Unblock Key) that was provided at the time of issuance. Do not attempt to force any more PIN entries — most tokens permanently brick after exceeding the PUK attempt limit as well.

PDFSign shows "No certificate found" even though my token manager shows the certificate

Run PDFSign as administrator (right-click shortcut, Run as administrator). Also ensure that the PKCS#11 DLL path is correctly detected — PDFSign auto-detects the standard installation paths for common middleware.
